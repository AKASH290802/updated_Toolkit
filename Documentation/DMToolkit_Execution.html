DM Toolkit Execution Guide

Step-by-Step Guide to Running the DM Toolkit

Step 1: Clone the Repository

Action:

Open Command Prompt and navigate to the C: drive.

Command:

git clone -b dm_tools --single-branch https://gitlab.tavant.com/kartik.kumar1/DMToolkitExecutor.git C:\DM_toolkit

Configuration:

Update linkedservice.json in the Services folder with required credentials or project-specific modifications.

Step 2: Install Dependencies

Action:

Navigate to the DM_toolkit directory in Command Prompt.

Command:

pip install -r requirements.txt

Details:

The requirements.txt file lists all necessary Python packages for the toolkit.

Step 3: Data Migration (Dataload Folder)

Scripts:

  • data_load2sql.py: Migrates data from Salesforce, Excel, or CSV to SQL Server.
  • dataframe.py: Migrates data from Salesforce or SQL to the local system.
  • dataloader.py: Migrates data from Excel, CSV, or SQL to Salesforce.

Pre-requisite:

Run mapping.py to generate mapping.json for the specific Salesforce object.

Output:

Source, error, success, and transformed files saved in the dataloader_logs folder.

Step 4: Mapping (Mapping Folder)

Action:

Run mapping.py to generate mapping for the selected Salesforce object.

Output:

Mapping saved in the mapping_logs folder.

Step 5: Unit Testing (Unit Testing Folder)

Status:

Module is in development but functional.

Action:

Run checks.py to generate a unit test case report based on Salesforce and object selection.

Output:

Report saved in the Unit Testing Generate folder.

Step 6: Data Validation (Data Validation Folder)

Validation Types:

  • Schema-level: Validates required fields, data types, date formats, email, phone, etc.
  • Custom-level: Uses custom error formulas to prevent invalid data entry in Salesforce.

Steps:

  1. Run Salesforce_details.py to generate a Salesforce object description locally.
  2. Run schema_validation.py to generate a CSV file for schema-level validation.
  3. Run Summary.py to generate a comprehensive data summary CSV, including duplicates and issues.

Custom Validation:

  • Run GenAI_Validation.py to create a Bundle folder with Python functions for custom validations.
  • Use Copilot to convert Salesforce error formulas into Python-based formulas in bundle.py.
  • Run validator.py to execute bundle.py and generate the custom-validated data file.

Step 7: Refer to Documentation

Resource:

Consult the Documentation folder for detailed PDFs, text files, and requirements.

Purpose:

Provides guidance on toolkit functionality, usage, troubleshooting, and additional configuration details.

Action:

Ensure all steps align with the documentation for smooth execution.

Summary

Key Steps:

  1. Clone the repository and configure credentials.
  2. Install dependencies using requirements.txt.
  3. Perform data migration with provided scripts.
  4. Generate mappings for Salesforce objects.
  5. Conduct unit testing and validate data.
  6. Refer to documentation for troubleshooting.

Next Steps:

Follow the documentation for any issues or advanced configurations.
