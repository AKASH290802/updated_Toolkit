DM Toolkit Execution Guide

Step-by-Step Guide to Running the DM Toolkit

Step 1: Clone the Repository

Action:

Open Command Prompt and navigate to the C: drive.

Command:

git clone -b dm_tools --single-branch https://gitlab.tavant.com/g.akash/toolkit.git

Step 2: Install Dependencies

Action:

Navigate to the DM_toolkit directory in Command Prompt Or In Terminal of the code Editor where you have the application, You can type

Command:

pip install -r requirements.txt

Details:

The requirements.txt file lists all necessary Python packages for the toolkit.

Step 3: Get application in your browser

Commands:

First open the saved Folder in any of the code Editor present in your laptop, then Type "streamlit run streamlit_app.py" in the Terminal of the code Editor, then the application gets opened in your browser

Step 4: Add your Org

Commands:

In the application go to Configuration tab and add your salesforce org by giving the credentials.

Credentials needed:

  • Username
  • Password
  • Security Token

Step 5: Perform Action

Commands:

Mandatory to do Data Validation

Validation Types:

  • Schema-level: Validates required fields, data types, date formats, etc.
  • Custom-level: Uses custom error formulas to prevent invalid data entry in Salesforce.
  • GenAI Validation: Converts the apex code of the Validation formulas to Python functions and will validates the data uploaded.

Step 6: Further Actions

Commands:

  • The data for a Object could be extracted(inserted few days back, all records or even can write a SOQL query).
  • The Valid Data only to be inserted(can be either inserted, updated or upserted).
  • The Unit Test Cases can be Generated for the selected Object.
  • Explore !!!

Summary

Key Steps:

  1. Clone the repository from GitLab.
  2. Install dependencies using requirements.txt.
  3. Run the application in your browser using Streamlit.
  4. Add your Salesforce org credentials in Configuration.
  5. Perform mandatory Data Validation (Schema, Custom, GenAI).
  6. Execute further actions: data extraction, insertion, unit testing.

Next Steps:

Explore the application and start your data validation journey!
